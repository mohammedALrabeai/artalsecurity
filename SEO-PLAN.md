# خطة تحسين محركات البحث — الموقع الرئيسي artalsecurity.com

> الهدف: تحويل الموقع الرئيسي إلى أداة جذب **عملاء** (منشآت وشركات) لخدمات الحراسات الأمنية
> المدنية والصناعية، والظهور في صدارة بحث جوجل والـ AI لكلمات نية تجارية.
> الجمهور: مدراء المنشآت، إدارات المشتريات، ملاك المشاريع — وليس الباحثين عن وظائف.

---

## 0) ملخص التشخيص (الوضع الحالي)

| العنصر | الحالة | الخطورة |
|--------|--------|---------|
| التصيير | React SPA عميل بالكامل — `build/index.html` صدفة فارغة (438 بايت، صفر محتوى) | 🔴 حرج |
| مكوّن `SEO.tsx` | موجود لكن **غير مستخدَم** في أي صفحة | 🔴 |
| اللغة الافتراضية | إنجليزية (`useState<Language>('en')`) | 🔴 |
| `robots.txt` / `sitemap.xml` | في `src/public/` ⇒ **لا تُنشَر** (Vite يتجاهلها)؛ والروابط أجزاء `#about` لا صفحات | 🔴 |
| الصفحات الفعلية | `/` و `/careers` فقط | 🟡 |
| الخدمات الـ8 | تقنية مساعدة (سيارات، كاميرات، إطفاء، تتبع...)؛ الحراسة المدنية/الصناعية غير بارزة كخدمة مستقلة | 🟡 |
| structured data | لا يوجد إطلاقاً | 🟡 |

**الاستنتاج:** الموقع مبني بشكل معاكس للـ SEO. الإصلاح التقني (المرحلة 0) شرط لنجاح كل ما بعده.

## 1) القرارات المعمارية (تحتاج موافقتك)

### أ) حل التصيير (الأهم)
الخيار الموصى به: **`vite-react-ssg`** — يولّد HTML ثابتاً مليئاً بالمحتوى لكل مسار عند البناء،
دون إعادة كتابة الموقع. البديل الأخف: `react-snap` (زاحف بعد البناء عبر Puppeteer).

| الخيار | المزايا | التكلفة |
|--------|---------|---------|
| **vite-react-ssg** (موصى) | HTML كامل لكل صفحة، تحكّم دقيق بالـ meta لكل مسار، الأنسب SEO | تعديل `main.tsx` و`App.tsx` لصيغة routes |
| react-snap | شبه بلا إعداد | أقل صيانةً، تحكّم أقل |

### ب) اللغة
- تحويل الافتراضي إلى **العربية** (السوق المستهدف سعودي).
- المرحلة الأولى: prerender النسخة العربية. الإنجليزية تبقى تبديلاً تفاعلياً.
- لاحقاً (اختياري): مسارات `/en/` مُسبقة التصيير لو أردنا SEO إنجليزي.
- `hreflang` للعربية/الإنجليزية في كل صفحة.

### ج) بنية الروابط المقترحة
```
/                                   الرئيسية (شركة حراسات أمنية في السعودية)
/services                            فهرس الخدمات
/services/manned-guarding            الحراسات الأمنية المدنية (الخدمة الأساسية)
/sectors/industrial                  حراسة المنشآت الصناعية والمصانع
/sectors/commercial-residential      حراسة المجمعات التجارية والسكنية
/sectors/construction-projects       حراسة المشاريع والمواقع الإنشائية
/sectors/government-vital            حراسة المنشآت الحكومية والحيوية (نفط/صحة/تعليم)
/locations/riyadh|jeddah|dammam|...  صفحات المدن
/blog/...                            مقالات B2B
/contact                             تواصل + طلب عرض سعر
/careers                             (قائم) فرع التوظيف
```
> ملاحظة: نتجنّب روابط الأجزاء `#about` نهائياً — أنشئ صفحات حقيقية فقط.

## 2) المرحلة 0 — الأساس التقني (حرج، أولاً)

**الهدف:** أن يرى جوجل/الـ AI/التواصل محتوى عربياً كاملاً في HTML الأولي.

**الملفات المتأثرة:**
- `package.json` — إضافة `vite-react-ssg` + سكربت بناء.
- `src/main.tsx` — التحويل لنقطة دخول SSG (`ViteReactSSG`).
- `src/App.tsx` — تصدير `routes` بدل `<BrowserRouter>` المباشر.
- `index.html` — عنوان/وصف عربي حقيقي + `lang="ar"` + `dir="rtl"` + OG + favicon.
- `src/context/LanguageContext.tsx` — الافتراضي `'ar'`.
- `src/components/SEO.tsx` — استخدامه فعلياً في كل صفحة (عنوان/وصف/canonical/hreflang لكل مسار).
- `vite.config.ts` — ضبط `publicDir` بحيث تُنشَر ملفات `public/` الصحيحة.
- نقل `robots.txt` و`sitemap.xml` إلى مجلد `public/` الذي يستخدمه Vite فعلياً.

**الخطوات:**
1. تركيب `vite-react-ssg` وتحويل التوجيه لصيغة `routes`.
2. لكل مسار: مكوّن `<SEO>` بعنوان/وصف فريدين.
3. إضافة structured data أساسي في الرئيسية: `Organization` + `SecurityService` + `LocalBusiness`
   (الاسم/العنوان/الهاتف/الخريطة + `sameAs` لقنوات التواصل) — يقوّي صندوق الـ AI بالاسم.
4. تحويل اللغة الافتراضية للعربية + `hreflang`.
5. إعادة بناء `sitemap.xml` بروابط الصفحات الحقيقية (سيُحدَّث مع كل صفحة جديدة).
6. `robots.txt` صحيح + سطر `Sitemap:`.

**معيار النجاح:** `curl https://artalsecurity.com/` يُظهر نصاً عربياً ووصفاً وstructured data
في HTML الخام (لا صدفة فارغة)، وتنجح أداة Rich Results Test.

## 3) المرحلة 1 — صفحات القطاعات (جوهر استهداف العملاء) 🎯

قالب موحّد لكل صفحة قطاع: H1 فريد + مقدمة (250–400 كلمة) + "لماذا نحن لهذا القطاع" +
أنواع المواقع المخدومة + المخاطر التي نعالجها + إشارات ثقة + **أسئلة شائعة** + زر **"اطلب عرض سعر"**.
كل صفحة: `Service` + `BreadcrumbList` + `FAQPage` schema.

### القطاعات الأربعة (كلها أولوية) والصفحات الفرعية:

**أ) الصناعي والمصانع — `/sectors/industrial`**
المصانع، المستودعات، المناطق اللوجستية، المدن الصناعية، المرافق الإنتاجية.
كلمات: `حراسات أمنية صناعية`، `حراسة مصانع`، `حراسة مستودعات`، `أمن المناطق الصناعية`.

**ب) التجاري والسكني — `/sectors/commercial-residential`**
المولات، المجمعات التجارية، الأبراج، الكمباوندات، الأحياء السكنية.
كلمات: `حراسات أمنية للمولات`، `أمن سكني`، `حراسة كمباوندات`، `حراسة مجمعات تجارية`.

**ج) المشاريع والإنشاءات — `/sectors/construction-projects`**
المواقع الإنشائية، المشاريع الكبرى، البنية التحتية، مواقع المقاولين.
كلمات: `حراسة مواقع إنشائية`، `أمن المشاريع`، `حراسة مشاريع المقاولات`.

**د) الحكومي والحيوي — `/sectors/government-vital`**
المنشآت الحكومية، النفطية والطاقة، الصحية (مستشفيات)، التعليمية (مدارس/جامعات).
كلمات: `حراسة منشآت حكومية`، `أمن المنشآت النفطية`، `حراسة مستشفيات`، `حراسة منشآت تعليمية`.

**+ صفحة الخدمة الأساسية — `/services/manned-guarding`**
"الحراسات الأمنية المدنية" (أفراد أمن مدرّبون، مسلح/غير مسلح) — الخدمة المحورية التي تربط كل القطاعات.

## 4) المرحلة 2 — صفحات المدن

`/locations/<slug>` لكل مدينة رئيسية (الرياض، جدة، الدمام، الخبر، مكة، المدينة، الجبيل، الطائف...).
"شركة حراسات أمنية في <المدينة>" + `LocalBusiness` schema لكل مدينة + ربط داخلي بصفحات القطاعات.
نفس بنية البيانات التي نجحت في فرع التوظيف (قائمة مدن مركزية تولّد الصفحات والـ sitemap والتذييل).

---

## 5) المرحلة 3 — تحويل الزائر إلى عميل (Conversion)

- **طلب عرض سعر** بارز في كل صفحة (نموذج قصير: الجهة/القطاع/المدينة/الهاتف) + واتساب + هاتف.
- إبراز إشارات الثقة الموجودة بالفعل في الكود: `CertificatesSection` (التراخيص),
  `ProjectsSection` (المشاريع), `TestimonialsSection` (آراء العملاء), شعارات العملاء.
- إضافة `AggregateRating`/`Review` schema لو توفّرت تقييمات حقيقية.
- صفحة/قسم "عملاؤنا" — حاسمة في قرار B2B.

---

## 6) المرحلة 4 — المحتوى والسلطة

- **مقالات B2B** تستهدف بحثاً أعلى في القمع وتغذّي ملخّصات الـ AI:
  - "كيف تختار شركة حراسات أمنية لمنشأتك"
  - "الفرق بين الحراسة الأمنية المدنية والصناعية"
  - "متطلبات تأمين المواقع الإنشائية"
  - "معايير اختيار شركة أمن للمولات والمجمعات"
- **رابط من فرع التوظيف** (jobs.artalsecurity.com) ← الموقع الرئيسي (تمرير سلطة؛ الفرع بدأ يكتسب زيارات).
- **Google Business Profile** + أدلّة الأعمال السعودية + المنصات القطاعية.

## 7) المرحلة 5 — القياس

- **Google Search Console** لنطاق الموقع الرئيسي + رفع الـ sitemap الجديد.
- تتبّع تحويلات "طلب عرض سعر" و"واتساب".
- مراجعة شهرية للكلمات/الانطباعات وتوسيع القطاعات/المدن حسب الأداء.

## 10) قرارات مفتوحة تحتاج ردّك

1. **حل التصيير**: `vite-react-ssg` (موصى) أم `react-snap`؟
2. **اللغة**: نجعل العربية الافتراضية ونؤجّل SEO الإنجليزي؟
3. **المدن**: نفس قائمة فرع التوظيف الـ17 أم قائمة أقصر تركّز على مدن العمليات الفعلية؟
4. **بيانات الكيان**: العنوان الرسمي + الهاتف + رابط الخريطة (لـ LocalBusiness وصندوق الـ AI).
5. **معلومات تجارية**: قائمة العملاء/المشاريع المسموح نشرها، والتراخيص (رقم ترخيص الأمن الصناعي إن وُجد).
